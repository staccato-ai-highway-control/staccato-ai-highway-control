# STACCATO VM 서버 구축 및 Git 협업 문서

STACCATO는 AI 기반 고속도로 정차 차량 탐지 및 교통 관제 시스템이다.  
본 문서는 MVP 시연을 위한 VM 서버 분리 구조, 서버별 역할, 프론트엔드 서버 구축 내용, GitHub 협업 방식을 정리한다.

---

## 1. 프로젝트 개요

STACCATO 프로젝트는 고속도로 주행 차로 및 갓길에 정차한 차량을 탐지하고, 탐지 결과를 기반으로 관리자에게 상황 정보를 제공하는 AI 기반 교통 관제 시스템이다.

MVP 단계에서는 다음 기능을 중심으로 구현한다.

- 관리자 웹 화면
- 로그인 / 회원가입
- CCTV 목록 및 관제 화면
- 이상상황 신고 및 조회
- 정차 차량 탐지 결과 확인
- LLM 기반 보고서 요약 및 생성
- 알림 및 관리자 기능
- Flask API 서버를 통한 DB, AI, ITS 연동

---

## 2. VM 서버 분리 구조

STACCATO 프로젝트는 최종 시연 환경을 고려하여 서비스를 VM 단위로 분리한다.

| 서버명 | IP | 담당 역할 |
|---|---:|---|
| AI-VM | `192.168.0.186` | AI 탐지, YOLO, OpenCV, LLM 관련 기능 |
| FLASK-VM | `192.168.0.187` | 메인 백엔드 API, 인증, DB 연동, AI/ITS 중계 |
| FRONTEND-VM | `192.168.0.188` | Next.js / React 관리자 웹 UI |
| ITS-VM | `192.168.0.189` | ITS API, CCTV, 교통 이벤트 연동 |
| DB-VM | `192.168.0.190` | MySQL DB 서버 |

---

## 3. 전체 아키텍처

```text
사용자 / 관리자
        ↓
FRONTEND-VM
192.168.0.188:3001
Next.js / React
        ↓
FLASK-VM
192.168.0.187:5000
Flask API / 인증 / DB 중계 / AI 중계 / ITS 중계
        ↓
DB-VM
192.168.0.190:3306
MySQL
```

AI 및 ITS 서버와의 통신은 프론트엔드에서 직접 처리하지 않는다.

```
FRONTEND-VM → FLASK-VM → DB-VM
FRONTEND-VM → FLASK-VM → AI-VM
FRONTEND-VM → FLASK-VM → ITS-VM
```

---

## 4. 서버별 역살 상세

**4-1. FRONTEND-VM**

| 항목    | 내용                |
| ----- | ----------------- |
| IP    | `192.168.0.188`   |
| 포트    | `3001`            |
| 실행 폴더 | `frontend-server` |
| 프레임워크 | Next.js / React   |
| 담당자   | 김도하               |

FRONTEND-VM은 관리자 웹 UI를 담당한다.

주요 기능은 다음과 같다.

- 로그인 / 회원가입 화면
- 대시보드 화면
- CCTV 관제 화면
- 이상상황 목록 및 상세 화면
- 게시판 화면
- LLM 보고서 화면
- 관리자 페이지
- Flask API 서버와 통신

프론트엔드는 DB, AI, ITS 서버에 직접 접근하지 않고 Flask API 서버를 통해서만 데이터를 요청한다.

**4-2. FLASK-VM**

| 항목    | 내용              |
| ----- | --------------- |
| IP    | `192.168.0.187` |
| 포트    | `5000`          |
| 프레임워크 | Flask           |
| 역할    | 백엔드 API 서버      |

FLASK-VM은 시스템의 중심 API 서버 역할을 한다.

주요 기능은 다음과 같다.

- 로그인 / 회원가입 API
- JWT 인증
- 사용자 권한 확인
- DB 서버 연동
- AI 서버 요청 중계
- ITS 서버 요청 중계
- 프론트엔드 요청 처리

**4-3. DB-VM**

| 항목    | 내용                        |
| ----- | ------------------------- |
| IP    | `192.168.0.190`           |
| 포트    | `3306`                    |
| DBMS  | MySQL                     |
| 실행 방식 | Docker 미사용, Ubuntu에 직접 설치 |

DB-VM은 Docker를 사용하지 않고 Ubuntu 서버에 MySQL을 직접 설치하여 운영한다.

보안상 DB 서버는 Flask 서버에서만 접근 가능하도록 설정한다.

```
허용 대상:
192.168.0.187 → 192.168.0.190:3306
```

Frontend 서버는 DB 서버에 직접 연결하지 않는다.

**4-4. AI-VM**

| 항목    | 내용                        |
| ----- | ------------------------- |
| IP    | `192.168.0.186`           |
| 역할    | AI 탐지 및 LLM 서버            |
| 주요 기술 | YOLO, OpenCV, Python, LLM |

AI-VM은 정차 차량 탐지 및 LLM 관련 기능을 담당한다.

주요 기능은 다음과 같다.

- 정차 차량 탐지
- CCTV 이미지 / 영상 분석
- 탐지 결과 생성
- LLM 보고서 생성 보조
- Flask 서버로 탐지 결과 전달

**4-5. ITS-VM**

| 항목 | 내용                     |
| -- | ---------------------- |
| IP | `192.168.0.189`        |
| 역할 | ITS / CCTV / 교통 이벤트 연동 |

ITS-VM은 외부 ITS API와 CCTV 정보를 연동하는 서버이다.

주요 기능은 다음과 같다.

- ITS API 연동
- CCTV 데이터 수집
- 교통 이벤트 데이터 수집
- 도로 위험도 보조 데이터 제공
